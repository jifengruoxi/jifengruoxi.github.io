## 为什么选择 Hugo + GitHub Pages

在信息爆炸的时代，拥有一个属于自己的独立博客，不仅是技术能力的展示，更是沉淀思考、记录成长的绝佳方式。经过一番调研，我最终选择了 **Hugo** 搭配 **GitHub Pages** 的方案。

这个组合的优势非常明显：
- **Hugo**：基于 Go 语言的静态网站生成器，构建速度极快，且通过 Markdown 写作，让我能专注于内容本身。
- **GitHub Pages**：提供免费、稳定的静态网页托管服务，配合 Git 版本控制，天然适合博客的迭代与备份。

## Hugo 的核心：Markdown 与 YAML

Hugo 的工作流非常符合程序员的直觉，它将内容与配置完美分离：

1. **Markdown 文档**：所有的文章、笔记都存放在 `content` 目录下，以 `.md` 格式编写。Hugo 会自动将其渲染为精美的 HTML 页面。
2. **YAML 配置**：网站的标题、菜单、主题、社交链接等全局设置，全部在 `config.yaml` 中定义。无需修改任何代码，只需调整配置文件的参数，网站风格即可焕然一新。

这种"配置即所得"的模式，极大地降低了维护成本，让我能随时根据心情或需求调整博客

## 总结与展望

从安装 Hugo、编写第一篇 Markdown，到配置 Git 并解决推送报错，整个过程虽然遇到了一些小插曲，但解决问题的过程本身就是最好的学习。
